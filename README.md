## サーバー構築手順（開発・参加者向け）

### 1. フォルダ準備とサーバー本体（JAR）の取得

- `mods/`、`plugins/`、およびサーバー本体の JAR は未完成の状態です
- サーバー本体の JAR は以下のリンクからダウンロードしてください  
  [https://drive.google.com/file/d/1VWP1QeHvMXf2QE0fSnYf-F1JEb_or6KN/view?usp=drive_link](https://drive.google.com/file/d/1VWP1QeHvMXf2QE0fSnYf-F1JEb_or6KN/view?usp=drive_link)
- `mods/`、`plugins/` の詳細は  
  各フォルダ内の `README.txt` を参照してください

---

### 2. サーバーの起動

- ルートディレクトリに起動用スクリプト `run.sh` を用意しています
- Java のパス、割り当てメモリ量、OS など  
  各自の環境に合わせて内容を編集してから実行してください
- 初回起動時に `eula.txt` が生成されます  
  内容を確認し、`eula=true` に書き換えてください

---

### 3. クライアント側（ゲーム起動）

- 以下のリンクから ZIP をダウンロードし、  
  Forge のインポート機能を利用してください  
  [https://drive.google.com/file/d/13K8TXeSkk0NFIpEP67gXa3wjMxSXG7qQ/view?usp=sharing](https://drive.google.com/file/d/13K8TXeSkk0NFIpEP67gXa3wjMxSXG7qQ/view?usp=sharing)
- サーバー側とは一部 MOD 構成が異なりますが、  
  接続に必要な設定はすべて含まれています

## 開発方針

導入する MOD・Plugin および全体コンセプトを段階的に確定させた後、  
Config（設定ファイル） や　KubeJS（スクリプト） の編集による  
本格的なカスタマイズを行う想定です。

## テスト時の起動構成

- CPU: AMD Ryzen 9 9900X3D
- メモリ: 64GB（DDR5）  
  - うち 24GB をサーバー専用に割り当て
- ディスク: NVMe SSD（Gen4）  
  - サーバー専用に 300GB 割り当て

※ 上記は検証時の環境です。
　同等以下の構成でも、小規模・検証用途であれば
　十分動作する可能性があります。
